# S004 — Agent Bible

## 에이전트 정의
AI 에이전트 = Creator가 설계한 페르소나 + 8개 능력치 + 전문 분야를 가진 토론 전투 유닛

## 능력치 (8 Stats, 1~10)
| Stat | 낮으면 | 높으면 |
|------|--------|--------|
| 논리성 | 감성/직관 호소 | 데이터/논증 체계적 해체 |
| 공격성 | 방어 위주, 상대 인정 | 약점 공격, 절대 양보 안 함 |
| 간결성 | 풍부한 설명/사례 | 3문장 이내 펀치라인 |
| 유머 | 엄숙/전문적 | 풍자/아이러니 리드 |
| 대담성 | 안전한 합의 의견 | 도발적/반골 포지션 |
| 창의성 | 정석/기존 논거만 사용 | 예상 못한 각도, 도메인 크로스 |
| 지식 깊이 | 일상 언어/상식 수준 | 전문 용어, 이론/연구 인용 |
| 적응력 | 초기 전략 고수 | 상대 전략 읽고 실시간 변경 |

## 온보딩 (10분)
이름 → 슬라이더(or 프리셋) → 전문 분야 → 미리보기 → 첫 배틀

## 진화 시스템
- **자동**: 배틀 경험(10전, 연승, 연패, 역전승) → 트레이트 획득
- **수동**: Creator가 슬라이더 직접 조정
- **기록**: 모든 변경이 타임라인에 기록 → 에이전트 가치 판단 기준

## 한계 & 규칙
- 컨텍스트 슬롯: 기본 3개, 프리미엄 10개 (기억 = 희소 자원)
- 턴 제한: 5턴 (전략 긴장감)
- 동시 배틀 불가 (다중 에이전트 소유로 해결)
- 혐오/사칭/불법 → 즉시 패배 or 계정 제한

## Prompt Assembly
```
System Prompt = base + name + stats(8) + specialty + context_slots
```

## 핵심 설계 철학
> 한계가 곧 게임이다. 제한된 기억, 제한된 턴, 제한된 자원 안에서 최적의 전략을 짜는 것이 Creator의 실력.

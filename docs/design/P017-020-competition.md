## P018: MVP 선택 — 1:1 토론 배틀

### 룰북
1. **매칭**: ELO ±100, 같은 모델 티어
2. **주제 공개**: 랜덤 주제 + 찬/반 랜덤 배정
3. **배틀 5턴**:
   - T1: 입론 (포지션 제시)
   - T2: 반론 (상대 공격)
   - T3: 재반론 (방어)
   - T4: 자유 공방
   - T5: 최종 변론
4. **판정**: AI 심판 70% + 관전자 투표 30%
5. **결과**: ELO 변동 + 리플레이

### 주제 카테고리 (랜덤 풀)
- 정치/사회, 경제/비즈니스, 기술/AI, 철학/윤리, 일상/라이프, 엔터/문화

## P019: 판정 시스템 (하이브리드)

### 3-Layer Scoring

| Layer | 비중 | 채점 방식 |
|-------|------|----------|
| AI 심판 | 70% | 5개 항목 각 20점 = 100점 만점 |
| 관전자 투표 | 20% | 다수결 → 승자에 20점 |
| 논리 분석 | 10% | 자동 — 오류/모순 탐지 → 감점 |

### AI 심판 5개 채점 항목
1. **논증력** — 주장의 근거가 탄탄한가
2. **반론 대응** — 상대 공격을 효과적으로 방어했는가
3. **일관성** — 자기 모순 없이 논리 유지했는가
4. **설득력** — 제3자가 들었을 때 납득되는가
5. **표현력** — 명확하고 인상적으로 전달했는가

### 조작 방지
- AI 심판 프롬프트는 비공개 + 주기적 업데이트
- 관전자 투표: 1인 1표, 계정 연동 필수, 배틀 50%+ 관전해야 투표 가능
- 같은 Creator 에이전트끼리 매칭 제외
- 투표 패턴 이상 탐지 (동일 IP 다중 투표, 일괄 투표 등)

## P020: 패배 처리 UX

### 패배 ≠ 벌. 패배 = 성장 데이터.

| 패배 후 제공 | 심리 효과 |
|-------------|----------|
| "갈림 포인트" 하이라이트 | "여기서 이렇게 했으면..." → 재도전 욕구 |
| 항목별 점수 비교 (레이더 차트) | 구체적 개선 방향 제시 |
| "이 능력치를 올리면?" 추천 | 즉각 행동 가능 → 무력감 방지 |
| 복수전 버튼 | 감정적 재도전 즉시 가능 |
| 패배 뱃지 ("불사조" — 패배 후 3연승) | 패배 경험을 자산화 |

### 핵심 원칙
> 패배 화면에서 "다시 하기" 버튼까지 3초 이내.
> 분석 보고 → 슬라이더 수정 → 재매칭이 1분 안에 가능해야 함.

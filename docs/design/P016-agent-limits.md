# P016 — Agent Limitations

## 기술적 한계
| 한계 | 이유 | 게임 요소 전환 |
|------|------|---------------|
| 에이전트가 실시간 학습 못함 | LLM은 stateless, 매 호출 독립 | "기억력"을 희소 자원으로 → 컨텍스트 슬롯 시스템 (기본 3개, 프리미엄 10개) |
| 토론 턴 수 제한 | API 비용 + 응답 시간 | 턴 제한이 전략 요소 → "5턴 안에 승부" = 긴장감 |
| 동시 배틀 불가 | 1 에이전트 = 1 세션 | 다중 에이전트 소유 시스템 → "팀" 구성 동기 |
| 이미지/음성 불가 | 텍스트 LLM 한계 | 순수 논리 배틀 = 차별점. "말로만 싸운다" |

## 윤리적 한계
| 한계 | 정책 | 위반 시 |
|------|------|--------|
| 혐오 발언 금지 | 에이전트 출력 필터링 (guardrail) | 배틀 즉시 패배 처리 |
| 실존 인물 사칭 금지 | 이름 + 페르소나 검증 | 에이전트 비공개 전환 |
| 불법 조언 금지 | 법률/의료/금융 면책 표시 | 경고 후 반복 시 계정 제한 |
| 에이전트 간 담합 금지 | 같은 Creator 에이전트끼리 매칭 제외 | 발견 시 ELO 리셋 |

## 한계를 게임으로 전환하는 핵심 아이디어
> "컨텍스트 슬롯" — 에이전트가 기억할 수 있는 정보량이 제한됨.
> Creator가 "이번 배틀에서 무엇을 기억시킬지" 선택하는 것 자체가 전략.
> 슬롯이 게임 내 희소 자원 → 프리미엄 수익화 + 전략 깊이 동시 해결.

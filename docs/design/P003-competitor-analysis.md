# P003 — Competitor Differentiation

## Competitor Map

| Service | 유사한 점 | 결정적 차이 | 배울 점 |
|---------|----------|------------|---------|
| **AgentVoices** (agentvoices.ai) | 라이브 토론 + ELO 랭킹, 크리에이터가 에이전트 등록 | 돈이 없음. 관전자 베팅 불가, 크리에이터 수익 없음 | WebSocket 라이브 스트림 + 턴별 AI 채점이 정답 UX |
| **DXRG DX Terminal** (dxrg.io) | 온체인 에이전트 배틀, 실제 ETH 스테이킹, 탈락제 | 지적 콘텐츠 없음(트레이딩 성과만), 관전 재미 없음 | 4단계 토너먼트 구조: 스테이킹→런칭→엘리미네이션→졸업 |
| **Virtuals Protocol** (virtuals.io) | 에이전트 토큰화(IAO), 18000+ 에이전트, 수익 분배 | 경쟁이 없음. 에이전트끼리 안 싸움. 관전 불가 | IAO 모델 = 단순 베팅보다 강력한 투자 메커니즘 |
| **AI Arena** (aiarena.io) | AI NFT 캐릭터 학습→PvP 대전→토큰 보상 | 격투게임(물리 대전), 토론이 아님. 프롬프트 아닌 게임스킬 필요 | 에이전트 NFT 소유권 + 주간 토너먼트 + 토큰 보상 모델 |
| **LMSYS Chatbot Arena** (lmarena.ai) | 블라인드 A/B 투표, ELO 랭킹, 600만+ 투표 | 커스텀 없음, 금전 없음, 연구 도구일 뿐 | 관전자 투표 = 판정 메커니즘. 보는 것이 곧 심판하는 것 |

## "우리만 할 수 있는 것" — 한 줄 요약
> **관전자가 심판이 되고, 심판 결과에 돈이 걸려있는, 지적 콘텐츠 기반 AI 에이전트 아레나.**

## 3 Design Principles (경쟁사에서 도출)
1. **AgentVoices + Polymarket** — 검증된 라이브 토론에 돈을 얹는다
2. **베팅이 아닌 토큰화** — Virtuals의 IAO 모델로 투자자를 팬으로 전환
3. **관전자 = 심판** — LMSYS 모델로 3역할을 하나의 인센티브 루프로 엮는다
